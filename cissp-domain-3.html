Security Architecture and Engineering

Building security in by design — secure engineering principles, the formal security models, information-system capabilities, the vulnerabilities of modern architectures, cryptography and cryptanalysis, and the physical security of sites and facilities.

Back to CISSP Notes

Sub-domains

Exam outline areas.

Domain 3 spans the engineering of security — ten outline areas from secure design principles and formal security models, through system capabilities and architecture vulnerabilities, to cryptography, cryptanalysis, and the physical security of sites and facilities.

3.1 — Secure design principles

Research, implement and manage engineering processes using secure design principles — least privilege, defense in depth, zero trust, secure defaults, privacy by design.

3.2 — Security models

The fundamental security models — Bell-LaPadula, Biba, Clark-Wilson, Brewer-Nash and the state machine / lattice foundations.

3.3 — Select controls

Select controls based upon systems security requirements — using evaluation criteria such as Common Criteria (EALs).

3.4 — Security capabilities of systems

Memory protection, TPM, secure boot, encryption/decryption, and trusted-computing capabilities.

3.5 — Assess & mitigate architecture vulnerabilities

Vulnerabilities of architectures and designs — client/server, databases, cloud, IoT, ICS/SCADA, virtualization, containers and serverless.

3.6 — Cryptographic solutions

Select and determine cryptographic solutions — symmetric, asymmetric, hashing, PKI, and key management.

3.7 — Cryptanalytic attacks

Methods of cryptanalytic attack — brute force, frequency analysis, known/chosen-plaintext, side-channel, and more.

3.8 — Site & facility design principles

Apply security principles to site and facility design — including CPTED and layered physical defense.

3.9 — Site & facility security controls

Design the controls — perimeter, power, HVAC, fire detection & suppression, and environmental protection.

3.10 — Information system lifecycle

Manage the information system lifecycle — from requirements and development through operation to secure disposal.
